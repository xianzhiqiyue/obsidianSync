## 2. 详细问题清单

### 2.1 高优先级问题 (🔴)

#### 🔴 1. 竞态条件风险 - 上传并发控制缺失

**位置**: `apps/obsidian-plugin/src/main.ts:274-287`

**问题代码**:
```typescript
for (const target of prepare.uploadTargets) {
  const snapshot = localPlan.hashToSnapshot[target.contentHash];
  if (!snapshot) {
    throw new Error(`missing local snapshot for hash ${target.contentHash}`);
  }
  await client.uploadObject(target.uploadUrl, snapshot.bytes, signal);
}
```

**问题描述**:
- 多个文件串行上传，网络延迟高时整体同步耗时过长
- 没有并发控制和错误隔离，单个上传失败可能导致整个同步中断
- 大文件上传时没有进度反馈和分片机制

**修复建议**:
```typescript
// 引入 p-map 或手写并发控制
import pMap from 'p-map';

const uploadResults = await pMap(
  prepare.uploadTargets,
  async (target) => {
    const snapshot = localPlan.hashToSnapshot[target.contentHash];
    if (!snapshot) {
      throw new Error(`missing local snapshot for hash ${target.contentHash}`);
    }
    await client.uploadObject(target.uploadUrl, snapshot.bytes, signal);
    return { success: true, hash: target.contentHash };
  },
  { concurrency: 3, signal }
);
```

**优先级**: P0

#### 🔴 2. 中断信号处理不完整

**位置**: `apps/obsidian-plugin/src/main.ts:254-324`

**问题描述**:
- `runSyncOnceInternal` 中的 `AbortSignal` 只传递给网络请求
- 本地文件操作（`writeLocalFile`, `deleteLocalFileIfExists` 等）没有检查信号
- 用户取消同步后，本地文件修改仍会继续执行

**影响**: 用户体验差，可能产生不一致的本地状态

**修复建议**:
```typescript
// 在关键操作前检查信号
private async writeLocalFile(path: string, bytes: ArrayBuffer, signal: AbortSignal): Promise<void> {
  if (signal.aborted) {
    throw new DOMException('Aborted', 'AbortError');
  }
  // ... 原有逻辑
}

// 在 applyRemoteChanges 循环中检查信号
for (const change of remoteChanges) {
  if (signal.aborted) {
    throw new DOMException('Aborted', 'AbortError');
  }
  // ... 处理变更
}
```

**优先级**: P1

#### 🔴 3. 潜在内存泄漏 - 全量下载到内存

**位置**: `apps/obsidian-plugin/src/main.ts:430-512`

**问题代码**:
```typescript
const bytesByHash: Record<string, ArrayBuffer> = {};
if (downloadHashes.length > 0) {
  const urls = await client.getDownloadUrls(...);
  for (const hash of downloadHashes) {
    const url = map.get(hash);
    bytesByHash[hash] = await client.downloadObject(url, signal); // 全部加载到内存
  }
}
```

**问题描述**:
- 同步大量文件时，所有文件内容同时加载到内存
- 可能导致 Obsidian 崩溃或系统内存不足

**修复建议**:
```typescript
// 分批处理，限制内存占用
const BATCH_SIZE = 10;
for (let i = 0; i < remoteChanges.length; i += BATCH_SIZE) {
  const batch = remoteChanges.slice(i, i + BATCH_SIZE);
  const batchHashes = batch.map(c => c.contentHash).filter(Boolean);

  // 只下载当前批次
  const urls = await client.getDownloadUrls(accessToken, this.settings.vaultId, batchHashes, signal);
  // 处理批次并释放内存
  await this.applyBatchChanges(batch, urls, signal);
}
```

**优先级**: P1

#### 🔴 4. 数据库缺少关键索引

**位置**: `apps/sync-api/src/routes/sync.ts:687-694`

**问题 SQL**:
```typescript
const result = await query<ChangeEventRow>(
  `SELECT checkpoint, op, file_id, path, version, content_hash
   FROM change_events
   WHERE vault_id = $1
     AND checkpoint > $2
   ORDER BY checkpoint ASC, created_at ASC
   LIMIT $3`,
  [params.vaultId, parsed.data.fromCheckpoint, limit + 1]
);
```

**问题描述**:
- `change_events` 表缺少 `(vault_id, checkpoint)` 复合索引
- 同步拉取时会进行全表扫描，数据量大时性能急剧下降

**修复建议**:
```sql
-- 添加复合索引
CREATE INDEX CONCURRENTLY idx_change_events_vault_checkpoint
ON change_events(vault_id, checkpoint ASC);

-- 如有必要，也添加覆盖索引
CREATE INDEX CONCURRENTLY idx_change_events_vault_checkpoint_cover
ON change_events(vault_id, checkpoint ASC)
INCLUDE (op, file_id, path, version, content_hash);
```

**优先级**: P0

### 2.2 中优先级问题 (🟡)

#### 🟡 5. 类型安全漏洞 - 强制类型转换

**位置**: `apps/sync-api/src/routes/sync.ts:246,666` 等多处

**问题代码**:
```typescript
const params = request.params as { vaultId: string }; // 强制类型转换
```

**问题描述**:
- Fastify 的 `request.params` 实际为 `unknown`，强制转换绕过类型检查
- 可能导致运行时错误

**修复建议**:
```typescript
import { z } from 'zod';

const paramsSchema = z.object({
  vaultId: z.string().uuid()
});

const parsed = paramsSchema.safeParse(request.params);
if (!parsed.success) {
  return reply.code(400).send({ code: 'INVALID_PARAMS', message: parsed.error });
}
const { vaultId } = parsed.data;
```

**优先级**: P2

#### 🟡 7. 错误信息可能泄露敏感信息

**位置**: `apps/sync-api/src/routes/sync.ts:655-658`

**问题代码**:
```typescript
return reply.code(409).send({
  code: "SYNC_COMMIT_FAILED",
  message: error instanceof Error ? error.message : "commit failed"
});
```

**问题描述**:
- 数据库错误、文件路径等内部信息可能暴露给客户端

**修复建议**:
```typescript
// 记录详细错误到日志
request.log.error({ err: error }, "sync commit failed");

// 返回通用错误给客户端
const isDev = appConfig.env === 'development';
return reply.code(409).send({
  code: "SYNC_COMMIT_FAILED",
  message: isDev ? error.message : "commit failed, please retry"
});
```

**优先级**: P2

#### 🟡 8. 事务执行时间过长

**位置**: `apps/sync-api/src/routes/sync.ts:425-647`

**问题描述**:
- commit 操作在一个大事务中处理所有 changes
- 大量变更时事务持有锁时间过长，影响并发

**建议**:
- 分批处理变更，每批提交一次
- 或者使用更低的隔离级别（如 `READ COMMITTED`）

**优先级**: P2
