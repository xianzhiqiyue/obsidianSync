# Obsidian 自建同步系统开发计划（v1）

## 1. 目标
在可控周期内交付“可稳定使用”的同步系统，而不是只可演示的 Demo。

## 2. 里程碑总览

| 阶段 | 周期 | 产出 |
| --- | --- | --- |
| P0 需求冻结 | 1 周 | 需求、架构、模块、API 文档评审通过 |
| P1 基础设施 | 1 周 | 项目骨架、数据库迁移、对象存储接入、鉴权框架 |
| P2 服务端同步闭环 | 2 周 | prepare/commit/pull 全链路可跑通 |
| P3 插件端同步闭环 | 2 周 | 首次同步、增量同步、冲突副本、重试机制 |
| P4 稳定性强化 | 2 周 | 指标日志告警、备份恢复、弱网与并发测试 |
| P5 Beta 发布 | 1 周 | 桌面/Android 安装文档、灰度发布、问题收敛 |

总周期建议：`9 周`（单人可延长到 10-12 周）。

## 3. 分阶段任务

## P0 需求冻结（Week 1）
- 任务：
  - 确认需求边界与非范围。
  - 冻结术语与数据模型。
  - 评审 API 契约与错误码。
- 完成标准：
  - 文档评审结论为“可开发”。
  - 高风险项有明确应对方案。

## P1 基础设施（Week 2）
- 服务端：
  - 初始化项目、配置管理、日志框架。
  - PostgreSQL schema 与迁移脚本。
  - MinIO/S3 SDK 封装。
  - JWT + refresh token + 设备注册。
- 插件端：
  - Obsidian 插件骨架、设置页、日志面板。
  - 本地状态存储（checkpoint/queue）。
- 完成标准：
  - 能完成登录、创 Vault、基础连通性检查。

## P2 服务端同步闭环（Week 3-4）
- 实现：
  - `sync/state`、`sync/prepare`、`sync/commit`、`sync/pull`。
  - CAS 校验、tombstone、rename/move 事件。
  - 幂等 key 处理与事务提交。
- 完成标准：
  - Postman/集成测试可验证完整上传下载闭环。

## P3 插件端同步闭环（Week 5-6）
- 实现：
  - Vault 扫描与变更生成。
  - 首次同步模式选择（Upload-First/Download-First）。
  - 增量同步计划与执行。
  - 冲突副本生成与错误提示。
- 完成标准：
  - macOS 与 Android 可双向同步，冲突不丢数据。

## P4 稳定性强化（Week 7-8）
- 实现：
  - 弱网重试、断网恢复、超时治理。
  - 指标、日志、告警。
  - 备份与恢复演练脚本。
  - 性能压测（并发设备、批量变更）。
- 完成标准：
  - 达到需求文档中的 NFR 与 AC 基线。

## P5 Beta 发布（Week 9）
- 实现：
  - 发布包与安装指南。
  - 灰度用户反馈收集渠道。
  - 高优先级缺陷修复。
- 完成标准：
  - Beta 用户可连续使用 1 周，无 P0 数据丢失问题。

## 4. 任务优先级（Must/Should/Could）

### Must
- 鉴权与设备撤销
- prepare/commit/pull 闭环
- tombstone + 冲突副本
- 重试与幂等
- 备份恢复

### Should
- 同步诊断面板
- 版本升级提醒
- 告警规则模板

### Could
- WebSocket 变更推送
- 大文件分块传输

## 5. 角色建议（最小配置）
- 1 名全栈工程师（可单人承担）
- 1 名测试/试用反馈（兼职）
- 1 名运维支持（可由工程师兼任）

## 6. 发布管理
- 分支策略：`main`（稳定）/`develop`（集成）/`release/*`。
- 版本号：语义化版本（`MAJOR.MINOR.PATCH`）。
- 发布节奏：每周一个可安装测试版。

## 7. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| Android 后台被杀 | 同步中断 | 前台触发优先 + 用户提示 + 自动续传 |
| 冲突频繁 | 用户困惑 | 冲突副本可视化 + 指南文档 + 指标监控 |
| 单机服务不稳定 | 数据风险 | 自动备份 + 演练 + 监控告警 |
| 升级兼容问题 | 客户端不可用 | API 版本管理 + 兼容字段策略 |

## 8. 完成定义（Definition of Done）
- 代码合并前通过单元与集成测试。
- 文档同步更新（需求/API/运维）。
- 在 macOS + Android 完成回归验证。
- 关键指标无回退，且无未评估高危缺陷。
