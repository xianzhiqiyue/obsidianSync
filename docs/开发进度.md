## P3 已完成项
- 插件端接入 API：
  - 登录 `/auth/login`
  - token 刷新 `/auth/token/refresh`
  - 拉取同步状态 `/vaults/:vaultId/sync/state`
  - 对接 `prepare/commit/pull/download-urls` 全链路
- 插件本地持久化改造：
  - 统一持久化结构（settings/state/auth），修复原有覆盖问题。
  - 增加 `fileIndexByPath` 用于维护 fileId/version/contentHash。
  - 增加登录、登出、单次同步命令和设置页快捷按钮。
- 插件同步主流程：
  - 扫描 vault 本地文件并基于 hash 生成 create/update/delete 变更。
  - 执行 prepare 后按需上传对象，再 commit。
  - pull 远端变更并应用到本地文件，同时回放更新本地索引。
  - 本地重命名/移动识别为显式 `rename/move`（不再全部退化为 delete+create）。
  - 收到 prepare 冲突后自动落地冲突副本文件。
  - 同步链路加入基础重试（指数退避，最多 3 次）。
  - 失败状态管理：
    - 记录失败队列、错误信息、连续失败次数。
    - 不可重试错误进入阻塞态，需要手动清理后再继续同步。
  - 失败队列恢复：
    - 队列持久化扩展为可重放载荷（`fileId/baseVersion/contentHash/attempts`）。
    - 同步前优先重放失败队列，并对陈旧任务做有效性裁剪后回退到实时扫描。
    - 旧版本本地状态自动兼容迁移到新队列结构。
  - 插件同步规划逻辑抽离为纯模块（`sync-planner`），并增加单元测试：
    - 失败队列有效重放。
    - 失效失败项自动裁剪并回退实时扫描。
    - rename/move 识别回归。
    - 旧队列格式兼容回归。
  - 前后台恢复与执行治理（移动端稳定性）：
    - 同步执行互斥与排队（运行中触发将合并为一次待执行任务）。
    - 移动端前台唤醒触发同步（`visibilitychange/focus`，含节流）。
    - 同步尝试超时治理（120s）+ `AbortSignal` 传递到所有 API 请求链路。
    - 非交互触发（定时/唤醒）默认静默失败，避免通知风暴。
  - 插件端集成回归（自动化）：
    - 新增 `sync-retry` 测试：弱网重试、部分上传失败恢复、超时中断后重试。
    - 新增 `state-store` 测试：不可重试失败阻塞态 + 手动清理恢复。
    - 新增 `sync-notify` 测试：交互/非交互通知阈值与冷却时间策略。
    - 回归测试命令统一为 `npm run --workspace @obsidian-sync/plugin test`（当前 13/13 通过）。
  - 通知策略与状态可视化：
    - 阻塞态通知按冷却时间节流（后台触发低噪声）。
    - 连续失败达到阈值后才进行后台通知（交互触发保持即时）。
    - 设置页新增运行状态面板（最近同步、结果、队列、失败统计）。
    - 重试控制抽象为 `sync-retry` 模块，主流程复用并保留状态记录。
